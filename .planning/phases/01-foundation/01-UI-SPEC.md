## Spacing Scale

Declared values (multiples of 4 only). All components use these token values exclusively via CSS custom property references or class names. Zero inline styles permitted.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-label gaps, inline chip padding |
| sm | 8px | Button internal horizontal gap, input icon padding |
| md | 16px | Default card padding, form field gap, list item spacing |
| lg | 24px | Card outer padding, section padding within Dashboard shell |
| xl | 32px | Between form steps in Onboarding, between card and viewport edge |
| 2xl | 48px | Vertical rhythm between major Auth card sections |
| 3xl | 64px | Full-page vertical centering offset for Auth screen |

Exceptions:
- Touch targets for interactive elements (nav tabs, buttons): minimum 44px tall to satisfy accessibility tap target guidelines. This does not break the 4pt grid — it is a floor constraint on height only.
- Navbar height: 56px (not in the token scale — this is a fixed structural dimension per DESIGN.md §2.1 layout inference and IMPLEMENTATION_PLAN.md Navbar.jsx spec).

## Typography

Two fonts declared. Four size roles. Two weights per font.

| Role | Font | Size | Weight | Line Height | Usage |
|------|------|------|--------|-------------|-------|
| Display | Outfit (`--font-display`) | 28px | 700 | 1.2 | Auth screen headline ("Welcome to ZenStudy"), Onboarding step headline |
| Heading | Outfit (`--font-display`) | 20px | 600 | 1.3 | Dashboard greeting ("Good morning, Aarav"), Onboarding section labels |
| Body | Inter (`--font-body`) | 16px | 400 | 1.5 | Input labels, helper text, dashboard placeholder descriptions |
| Label / UI | Inter (`--font-body`) | 14px | 500 | 1.4 | Button text, nav tab labels, form field captions, badge text |

Rules:
- Outfit is reserved for display and headings only. Do not use Outfit for body copy or input placeholder text.
- Inter is used for all functional UI copy: labels, captions, error messages, input placeholders.
- No font size outside the four declared sizes. Outfit: weights 600, 700 only. Inter: weights 400, 500 only.
- Minimum rendered font size: 14px (never smaller — accessibility constraint).

Source: DESIGN.md §1 (`--font-display: 'Outfit'`, `--font-body: 'Inter'`).

**Accent reserved for (Phase 1 only):**
- `--accent-violet`: primary CTA button background (`btn-primary`), active tab underline indicator, focused input outline, Auth card border in Signup mode
- `--accent-teal`: Auth card border shift in Login mode (DESIGN.md §2.1: "container shifts borders from purple (signup) to teal (login) dynamically"), onboarding final step success indicator

**Glassmorphic surface contract:**
- All cards use `.glass-card` utility: `background: var(--bg-card); backdrop-filter: var(--glass-blur); -webkit-backdrop-filter: var(--glass-blur); border: 1px solid var(--border-glass); border-radius: 16px;`
- Frosted edge gradient: `border-image` or `::before` overlay using `linear-gradient(135deg, rgba(255,255,255,0.1), rgba(255,255,255,0.02))` per DESIGN.md §1 aesthetic guidelines.
- `--glass-blur` value: `blur(14px)`. Never animate `backdrop-filter` directly — animate `opacity` or `transform` instead (RESEARCH.md anti-pattern).

**Pulse animation for living canvas:**
- Background or large accent blobs: `animation: pulse 8s infinite` per DESIGN.md §1.
- `@keyframes pulse`: `0%, 100% { opacity: 0.6; } 50% { opacity: 1; }` — defined in `src/index.css`.

Source: DESIGN.md §1 (all raw token values), DESIGN.md §2.1 (border color shift), RESEARCH.md Pattern 5.

## Surface-by-Surface Interaction Contract

### Surface 1: Auth Screen (`Auth.jsx`)

**Layout:** Full-viewport `--bg-deep` background. Centered single `.glass-card` (max-width: 420px). No scroll.

**States:**
- Default: Signup tab active. Card border glows `--accent-violet`.
- Login tab active: Card border transitions to `--accent-teal` (CSS `transition: border-color 0.3s ease`).
- Guest entry: No tab state change — immediate action.

**Tabs:** "Sign Up" / "Sign In" — toggle controls form fields shown. No routing change, pure conditional render.

**Form fields (Signup):**
- Email input
- Password input
- Name input (shown only on Signup tab)
- All inputs: `background: rgba(255,255,255,0.05); border: 1px solid var(--border-glass); color: var(--text-main); border-radius: 8px; padding: 12px 16px;`
- Focus state: `outline: 2px solid var(--accent-violet); outline-offset: 2px; border-color: transparent;` — no additional glow animation (GPU cost)

**Buttons:**
- "Create Account" / "Sign In": `.btn-primary` (violet background, Outfit 14px weight 500, 44px min-height, 12px border-radius, `padding: 12px 24px`). On `:active`: `transform: translateY(2px) scale(0.98)`.
- "Enter as Guest": Secondary style — `background: transparent; border: 1px solid var(--border-glass); color: var(--text-muted);` Same size and active transform. On hover: `border-color: var(--border-glow); color: var(--text-main);`

**Error state:** Inline below the triggering field. Inter 14px weight 400, color `--color-destructive`. Text: see Copywriting Contract.

**Loading state:** Button disabled, label replaced with "…" (ellipsis). No spinner component — text-only to minimize complexity.

---

### Surface 2: Onboarding ("Zen Gate") (`Onboarding.jsx`)

**Layout:** Full-viewport `--bg-deep` background. Centered `.glass-card` (max-width: 480px). Step progress indicator at top. No back button on Step 1; "Back" text link on Steps 2–4.

**Steps (4 total, matching AUTH-04):**

| Step | Content | Interaction |
|------|---------|-------------|
| 1 | Name input + Target Exam selection (JEE / NEET / UPSC / Boards) | Text input + 4 selectable pill buttons |
| 2 | Exam Target Date | `<input type="date">` — native date picker, styled to match glass theme |
| 3 | Choose Starting Plant Seed (Cactus / Bonsai / Fern) | 3 interactive SVG cards, pulsating with `animation: breathe 4s ease-in-out infinite` |
| 4 | Confirmation summary | Read-only display of choices, "Begin Your Journey" CTA |

**Step indicator:** 4 dots, active dot filled `--accent-violet`, inactive dots `--border-glass`. 8px diameter circles with 8px horizontal gap.

**Exam pill buttons (Step 1):**
- Unselected: `background: rgba(255,255,255,0.05); border: 1px solid var(--border-glass); border-radius: 8px; padding: 10px 20px;`
- Selected: `background: rgba(139, 92, 246, 0.15); border: 1px solid var(--accent-violet); color: var(--text-main);`
- Transition: `border-color 0.2s ease, background 0.2s ease`

**Plant seed cards (Step 3):**
- 3 cards side by side (flex row, `gap: 16px`). Each: 120px × 140px `.glass-card` containing an inline SVG placeholder (actual SVG rendering is Phase 2 — placeholder is a simple geometric shape with the plant name label below).
- Selected card: `border-color: var(--accent-violet); box-shadow: 0 0 12px var(--border-glow);`
- Each card has the `breathe` keyframe on its SVG element (scale 1 → 1.08 → 1 over 4s) per DESIGN.md §2.2.

**"Continue Setup" CTA (Steps 1–3):** `.btn-primary` full-width. Disabled (opacity 0.4, pointer-events none) until required field(s) for current step are filled.

**"Back" link (Steps 2–4):** Positioned top-left of card. Inter 14px weight 400, `color: --text-muted`. On hover: `color: --text-main`. Clickable area padded to 44px height.

**Transition between steps:** Fade out current card body (`opacity: 0, transform: translateX(-8px)`) then fade in next (`opacity: 1, transform: translateX(0)`). Duration: 200ms ease. Steps render in the same card container — only the inner content animates.

### Surface 3: Navbar (`Navbar.jsx`)

**Layout:** Fixed top. Full viewport width. Height: 56px. `background: var(--bg-card); backdrop-filter: var(--glass-blur); -webkit-backdrop-filter: var(--glass-blur); border-bottom: 1px solid var(--border-glass);` — frosted nav strip.

**Left slot:** App name "ZenStudy" — Outfit 20px weight 600, `--text-main`. No logo image.

**Center slot (on Dashboard):** Nav tab strip for feature areas: Vent · Garden · League · Bazaar · Insights. Phase 1 renders these as inert placeholders (visually present, click logs a `console.info` in dev mode). Active tab: `--accent-violet` underline indicator (2px bottom border). Inactive: `--text-muted` Inter 14px weight 500. Tab min-width: 44px.

**Right slot:** User greeting snippet (first name, Outfit 14px weight 500, `--text-main`) + streak counter badge (fire icon from lucide-react + number, `--accent-violet` text) + ZP balance badge (star icon from lucide-react + number, `--text-muted`). Reading from profile — zero/initial values acceptable in Phase 1.

**Semantic element:** `<header role="banner">` containing `<nav aria-label="Main navigation">` for the tab strip.

**Accessibility:** Each tab is a `<button>` (not `<a>`) since routing is state-machine based. Active tab has `aria-current="page"`. Tab strip has `role="tablist"` if implemented as tabs, or `role="navigation"` if as nav links. Nav must be keyboard-navigable (Tab key cycles tabs, Enter/Space activates).

---

### Surface 4: Dashboard Shell (`Dashboard.jsx`)

**Layout:** Below fixed Navbar (56px offset via `padding-top: 56px` or CSS grid `grid-template-rows: 56px 1fr`). `--bg-deep` base. Max-width 1024px centered content area with `padding: 24px`.

**Greeting section:** Full-width. Outfit 28px weight 700 (`--text-main`): "Good [time-of-day], [firstName]." Inter 16px weight 400 (`--text-muted`) subtitle: "Your wellness journey continues." Time-of-day derived from local hour: morning 5–11, afternoon 12–16, evening 17–20, night 21–4.

**Feature placeholder panels (5 total):** One per nav tab (Vent Rooms, Mind Garden, Zen League, Zen Bazaar, Insights). Each is a `.glass-card` with `padding: 24px`, displaying:
- Panel title: Outfit 20px weight 600 (`--text-main`)
- Placeholder body: Inter 16px weight 400 (`--text-muted`) — see Copywriting Contract for exact copy per panel.
- No CTA buttons in Phase 1 placeholders — these are replaced in Phases 2–4.

**Active panel visibility:** Only the panel matching the active nav tab is rendered. Others are not mounted (conditional render, not CSS `display: none`) to keep Phase 1 DOM clean.

**Mock mode indicator:** When `MOCK_MODE === true`, render a small pill banner at top of Dashboard: `background: rgba(24, 94, 50%, 0.15); border: 1px solid var(--accent-teal); border-radius: 20px; padding: 4px 12px;` with Inter 12px weight 500 text (color `--accent-teal`): "Demo mode — data stored locally."  This helps examiners understand offline mode without disrupting the UI.

## Animation Catalog (Phase 1 scope only)

All keyframes defined once in `src/index.css`. Components reference them by name only.

| Keyframe | Definition | Applied to |
|----------|------------|------------|
| `breathe` | `0%,100% { transform: scale(1); opacity: 0.8; } 50% { transform: scale(1.08); opacity: 1; }` | Plant seed SVG cards in Onboarding Step 3. Duration: 4s, `ease-in-out`, `infinite`. |
| `pulse` | `0%,100% { opacity: 0.6; } 50% { opacity: 1; }` | Ambient background accent blobs (optional decorative layer). Duration: 8s, `ease-in-out`, `infinite`. |
| `incinerate` | `0% { opacity:1; transform:translateY(0) scale(1); } 100% { opacity:0; transform:translateY(60px) scale(0.6); filter:blur(4px); }` | Phase 2 (Burn Chamber). Defined now, not applied in Phase 1. |
| `matrix-fall` | `0% { opacity:1; transform:translateY(0); } 100% { opacity:0; transform:translateY(100px); letter-spacing:4px; }` | Phase 2 (Rant Room). Defined now, not applied in Phase 1. |

**Rule:** Do not animate `backdrop-filter`. Animating GPU compositing properties causes jank. Use `opacity` and `transform` exclusively for motion.

Source: DESIGN.md §1, IMPLEMENTATION_PLAN.md Phase 1, RESEARCH.md Pattern 5 and Anti-Patterns.

## Accessibility Contract

Mandatory for all Phase 1 surfaces. Based on IMPLEMENTATION_PLAN.md §1 (Accessibility parameter) and ROADMAP.md Phase 4 success criterion 5.

| Requirement | Implementation |
|-------------|----------------|
| Semantic HTML5 | `<header>`, `<main>`, `<nav>`, `<button>`, `<form>`, `<label>`, `<input>` — no `<div>` for interactive elements |
| Focus visibility | All interactive elements: `:focus-visible { outline: 2px solid var(--accent-violet); outline-offset: 3px; }` — defined globally in `src/index.css`. Never `outline: none` without a visible replacement. |
| ARIA on Navbar tabs | `role="tablist"` on tab container, `role="tab"` on each tab, `aria-selected="true/false"`, `aria-current="page"` on active tab |
| ARIA on Onboarding | `aria-describedby` linking step description to step indicator; `aria-live="polite"` region for validation errors |
| ARIA on Auth | Error messages: `role="alert"` so they are announced immediately on appearance |
| Touch targets | All interactive elements: minimum 44 × 44px clickable area |
| Color contrast | `--text-main` (`hsl(210, 40%, 98%)`) on `--bg-deep` (`hsl(224, 32%, 8%)`): contrast ratio ~18:1 — passes WCAG AAA. `--text-muted` (`hsl(215, 20%, 65%)`) on `--bg-deep`: ~5.5:1 — passes WCAG AA. `--accent-violet` text on `--bg-deep`: verify at implementation; if insufficient, use `--text-main` with `--accent-violet` as background. |
| No `dangerouslySetInnerHTML` | All user-supplied data (name, exam label) rendered as React text nodes only |

## Implementation Constraints

These constraints flow from CONTEXT.md decisions and must not be overridden by planner or executor:

| Constraint | Source | Rule |
|------------|--------|------|
| No Tailwind | D-03 | All styles in `src/index.css` as CSS custom properties and utility classes. No utility-first class names from any Tailwind-like library. |
| No inline styles | IMPLEMENTATION_PLAN.md §1 | `style={{}}` props are forbidden. Use `className` with classes from `src/index.css`. |
| No `dangerouslySetInnerHTML` | IMPLEMENTATION_PLAN.md §1, RESEARCH.md | All dynamic text is React text nodes. |
| No `react-router-dom` | D-04 | Routing is `(user, profile)` state machine in `App.jsx` conditional render. |
| No shadcn | D-03 | No `components.json`, no `@radix-ui` imports, no shadcn primitives. |
| Lucide icons only | D-02 | All icons from `lucide-react`. No SVG files for icons, no other icon library. |
| Outfit + Inter only | DESIGN.md §1 | No other web fonts. Google Fonts CDN in `index.html` with `font-display: swap`. |
| HSL tokens only | DESIGN.md §1 | No hardcoded hex, rgb, or hsl literals in component CSS classes or inline styles. All values via CSS custom properties. |
| `backdrop-filter` prefix | RESEARCH.md Pitfall 6 | Always pair `-webkit-backdrop-filter` with `backdrop-filter` on every glass element. |
| try/catch on localStorage | RESEARCH.md Pitfall 3 | Every `localStorage.setItem` call is wrapped in try/catch. |
